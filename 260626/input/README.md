# CSV JOIN練習用データ

日本国内のEC事業を想定した合成データです。実在する個人情報は含まれていません。

## ファイルとリレーション

| ファイル | 行数 | 主キー | 外部キー |
|---|---:|---|---|
| `顧客.csv` | 350 | `顧客ID` | - |
| `注文.csv` | 720 | `注文ID` | `顧客ID` → 顧客 |
| `注文明細.csv` | 980 | `注文明細ID` | `注文ID` → 注文 |
| `問い合わせ.csv` | 520 | `問い合わせID` | `顧客ID` → 顧客、`注文ID` → 注文 |

リレーション:

```text
顧客 1 ── N 注文 1 ── N 注文明細
 │            │
 └── N 問い合わせ ──┘
```

`問い合わせ.csv`の`注文ID`は、注文に関係しない一般的な問い合わせの場合は空欄です。空欄でない外部キーは、すべて実在する親レコードに接続できます。

## 練習問題

1. 顧客ごとの注文数を`注文回数`列として追加する。
2. 顧客ごとの問い合わせ数を`問い合わせ回数`列として追加する。
3. 注文ごとの明細行数と合計商品数量を集計する。
4. 注文ごとに問い合わせの有無を`問い合わせ有無`として追加する。
5. 注文履歴はあるが、問い合わせ履歴がない顧客を抽出する。
6. 顧客ごとの累計購入金額、平均注文金額、最終注文日を計算する。
7. 商品カテゴリごとのユニーク注文数と売上合計を計算する。
8. `LEFT JOIN`を使い、注文履歴のない顧客と問い合わせのない注文を抽出する。
9. 顧客 → 注文 → 注文明細を結合し、JOINによって行数が増える理由を確認する。
10. 顧客ごとの注文回数と問い合わせ回数から問い合わせ率を計算する。

## pandasの例

```python
import pandas as pd

customers = pd.read_csv("顧客.csv")
orders = pd.read_csv("注文.csv")

order_counts = (
    orders.groupby("顧客ID")
    .size()
    .rename("注文回数")
    .reset_index()
)

result = customers.merge(order_counts, on="顧客ID", how="left")
result["注文回数"] = result["注文回数"].fillna(0).astype(int)
```

CSVはExcelでも日本語が文字化けしにくいUTF-8 BOM形式です。金額は日本円を想定しています。
